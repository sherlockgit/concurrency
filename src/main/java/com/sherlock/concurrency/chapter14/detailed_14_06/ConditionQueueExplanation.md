# 什么是条件队列

## 问题

根据 `14-6` 的 `BoundedBuffer` 示例，通俗易懂地说下什么是条件队列。

## 简单理解

条件队列可以理解成：

```text
一批线程因为“条件还不满足”，暂时排队等待的地方。
```

在 Java 的内置锁机制中，每个对象都可以作为一把锁。每个对象也都自带一个等待区域，调用这个对象的 `wait()` 后，线程就会进入这个对象对应的条件队列。

在 `14-6` 里，`BoundedBuffer` 使用的是 `this` 这把锁。所以：

```java
wait();
notifyAll();
```

实际等价于：

```java
this.wait();
this.notifyAll();
```

也就是说，线程等待的是 `this` 这个对象上的条件队列。

## 用 BoundedBuffer 理解

`BoundedBuffer` 有两个状态条件：

| 操作 | 需要满足的条件 |
| --- | --- |
| `put` | 缓冲区未满 |
| `take` | 缓冲区非空 |

消费者调用 `take()` 时，如果缓冲区为空：

```java
while (isEmpty()) {
    wait();
}
```

这段代码的意思是：

```text
我想 take，但是缓冲区现在是空的。
条件不满足，我先进入 this 的条件队列里等待。
同时我释放 this 这把锁，让生产者有机会进来 put。
```

生产者调用 `put()` 放入元素后：

```java
doInsert(value);
notifyAll();
```

这段代码的意思是：

```text
我已经放入元素了。
缓冲区可能从“空”变成了“非空”。
那些在条件队列里等待的线程，可以醒来重新检查条件了。
```

## notifyAll 不是直接交接执行权

`notifyAll()` 不是直接把元素交给消费者，也不是保证消费者马上执行。

它只是发出一个通知：

```text
共享状态可能变了，你们醒来重新检查一下自己的条件。
```

被唤醒的线程还要重新竞争 `this` 锁。只有重新拿到锁之后，才会从 `wait()` 返回。

所以消费者被唤醒后，还要重新执行条件检查：

```java
while (isEmpty()) {
    wait();
}
```

如果缓冲区真的不空，才会继续执行：

```java
return doExtract();
```

## 为什么 wait 会释放锁

如果消费者发现缓冲区为空，但它一直拿着锁不放，那么生产者就进不来，也就没法放入元素。

所以 `wait()` 的关键作用是：

```text
当前线程进入条件队列等待，同时释放锁。
```

这样其他线程才能进入同步代码块，修改共享状态。

在 `BoundedBuffer` 中：

1. 消费者发现缓冲区为空。
2. 消费者调用 `wait()`，释放锁并等待。
3. 生产者获得锁，执行 `put()`。
4. 生产者放入元素，调用 `notifyAll()`。
5. 消费者被唤醒，重新竞争锁。
6. 消费者拿到锁后，重新检查 `isEmpty()`。
7. 如果缓冲区非空，消费者取出元素。

## 生活类比

可以把 `BoundedBuffer` 想象成一个仓库：

| 概念 | 类比 |
| --- | --- |
| 缓冲区 | 仓库 |
| 生产者 | 放货的人 |
| 消费者 | 取货的人 |
| 锁 | 仓库门钥匙 |
| 条件队列 | 仓库门口的等待区 |

仓库为空时，消费者不能一直占着仓库门不放。否则放货的人进不来，仓库永远不会有货。

正确做法是：

```text
消费者发现没货，就去等待区等，并把钥匙交出来。
生产者拿到钥匙，进仓库放货。
生产者放完货后，通知等待区的人：仓库状态变了，可以再来看看。
消费者醒来后重新抢钥匙，进仓库检查是否真的有货。
```

这个“等待区”就是条件队列。

## 条件队列等待的是“条件”，不是“锁”

容易混淆的一点是：

```text
线程进入条件队列，不是因为拿不到锁，而是因为拿到锁之后发现条件不满足。
```

拿不到锁时，线程是在等待进入 `synchronized`。

拿到锁以后，如果发现共享状态不满足要求，比如缓冲区为空，就调用 `wait()` 进入条件队列。

所以有两种等待：

| 等待类型 | 等待原因 |
| --- | --- |
| 等待锁 | 想进入 `synchronized`，但锁被别人持有 |
| 等待条件 | 已经进入 `synchronized`，但发现条件不满足，于是调用 `wait()` |

条件队列解决的是第二种问题。

## 一句话总结

条件队列就是：

```text
线程在拿到锁后发现条件不满足，于是调用 wait() 释放锁并进入等待的地方。
等其他线程改变状态并调用 notifyAll() 后，它们再醒来重新检查条件。
```
